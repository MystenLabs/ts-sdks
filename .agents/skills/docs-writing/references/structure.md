## meta.json Format

### Root Package meta.json

Every top-level package directory must have a `meta.json` with `root: true`:

```json
{
	"root": true,
	"title": "Sui SDK",
	"description": "TypeScript interfaces for Sui",
	"pages": ["index", "install", "hello-sui", "clients", "transaction-building", "cryptography"]
}
```

Fields:

- `root` (required for top-level): Set to `true` to mark as a root section
- `title` (required): Display name for the section
- `description` (recommended): One-line description used in LLM index
- `pages` (required): Ordered array of page names (without `.mdx` extension) and subdirectory names

### Subsection meta.json

Subdirectories can have their own `meta.json`:

```json
{
	"title": "Clients",
	"pages": ["index", "core", "grpc", "graphql", "json-rpc"]
}
```

Subsection meta.json does not have `root: true`.

### Special page names

- `"..."` тАФ Fumadocs rest pages marker. Includes all remaining MDX files not explicitly listed. Used sparingly.
- `"index"` тАФ Maps to `index.mdx` in the directory. Appears first in navigation.

### Pages array ordering

The `pages` array controls navigation order. Put the most important/foundational pages first:

1. `index` (overview/introduction)
2. Getting started / setup pages
3. Core concepts
4. Advanced topics
5. Migration guides (last)

## Build Artifacts

### Combined docs (docs package)

Generated by `pnpm --filter @mysten/docs build:docs` (runs `tsx scripts/build-docs.ts --all`):

```
packages/docs/dist/
тФЬтФАтФА llms-index.md           # Combined routing index with all pages
тФЬтФАтФА sui/
тФВ   тФЬтФАтФА llms-index.md       # Per-section index (same as packages/sui/docs/llms-index.md)
тФВ   тФЬтФАтФА index.md
тФВ   тФЬтФАтФА clients/
тФВ   тФВ   тФЬтФАтФА index.md
тФВ   тФВ   тФЬтФАтФА core.md
тФВ   тФВ   тФФтФАтФА ...
тФВ   тФФтФАтФА ...
тФЬтФАтФА dapp-kit/
тФФтФАтФА ...
```

### Per-package docs (SDK packages)

Each SDK package with docs has `"build:docs": "tsx ../docs/scripts/build-docs.ts"` in its package.json. This generates a `docs/` directory inside the package:

```
packages/sui/docs/
тФЬтФАтФА llms-index.md           # Per-package index
тФЬтФАтФА index.md
тФЬтФАтФА clients/
тФВ   тФЬтФАтФА index.md
тФВ   тФФтФАтФА ...
тФФтФАтФА ...
```

Per-package docs are included in the published npm package via the `files` array. They are NOT committed to git (`packages/*/docs/` is in `.gitignore`).

The combined docs (`--all` mode) apply `EXCLUDED_SUBDIRS` (e.g. `dapp-kit/legacy`) to skip content not relevant to the main index. Per-package docs include ALL content for that package (no exclusions).

### Build ordering

Turbo ensures `build:docs` runs before `build` for each package. The `build:docs` task only reads MDX source files тАФ it does not depend on any built SDK packages.

## Adding a New Package Section

1. Create the content directory: `packages/docs/content/my-package/`
2. Create `meta.json` with `root: true`, `title`, `description`, and `pages`
3. Create `index.mdx` with `title` and `description` frontmatter
4. Add additional pages as `.mdx` files
5. List all pages in `meta.json` `pages` array
6. Add `"build:docs": "tsx ../docs/scripts/build-docs.ts"` to `packages/my-package/package.json` scripts
7. Add `"docs"` to the `files` array in `packages/my-package/package.json`
8. Add `@mysten/my-package` as a dependency in `packages/docs/package.json` (for typedoc API reference generation)
9. Regenerate: `pnpm --filter @mysten/docs build:docs`
10. Validate: `pnpm --filter @mysten/docs validate-docs`

## Adding a Subsection

1. Create the subdirectory within an existing package directory
2. Create `meta.json` with `title` and `pages`
3. Add the subdirectory name to the parent `meta.json` `pages` array
4. Create MDX files for each page listed in `pages`
5. Regenerate and validate
