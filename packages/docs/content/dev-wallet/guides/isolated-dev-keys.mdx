---
title: Isolated Dev Keys
---

Keep your development keys completely separate from your mainnet wallet.

## The problem

You don't want to connect your real wallet — the one with actual SUI — to a development app.
Accidental transactions, buggy code, or a misclick could cost real money. And production wallets mix
dev and real accounts in the same interface.

## Solution

Dev Wallet runs entirely separate from your production wallet. Depending on the adapter you choose,
your dev keys are either ephemeral or scoped to the browser:

### Ephemeral keys (InMemory)

Keys exist only in JavaScript memory. They're gone the moment the page reloads. Nothing is persisted
anywhere.

```typescript
import { InMemorySignerAdapter } from '@mysten/dev-wallet/adapters';

const adapter = new InMemorySignerAdapter();
await adapter.initialize();
await adapter.createAccount({ label: 'Throwaway' });
// This key cannot touch your real wallet
```

### Browser-scoped keys (WebCrypto)

Keys are stored in IndexedDB via the Web Crypto API. They persist across page reloads but are:

- **Non-extractable** — private keys cannot be exported from the browser's crypto engine
- **Origin-scoped** — only your dev server can access them
- **Browser-scoped** — they don't exist outside this browser profile

```typescript
import { WebCryptoSignerAdapter } from '@mysten/dev-wallet/adapters';

const adapter = new WebCryptoSignerAdapter();
await adapter.initialize();
await adapter.createAccount({ label: 'Dev Only' });
// Keys are locked to this browser, this origin
```

## Security boundary

| Concern                             | InMemory            | WebCrypto                      |
| ----------------------------------- | ------------------- | ------------------------------ |
| Keys persist across reload          | No                  | Yes (IndexedDB)                |
| Keys exportable                     | No (in memory only) | No (non-extractable CryptoKey) |
| Keys accessible from other origins  | No                  | No                             |
| Keys accessible from other browsers | No                  | No                             |
| Risk to mainnet funds               | None                | None                           |

Neither adapter can access your production wallet's keys, and your production wallet cannot access
Dev Wallet's keys.

## Production warning

Dev Wallet emits a console warning if used in a production build:

```
[@mysten/dev-wallet] DevWallet is intended for development only. Do not use in production.
```
