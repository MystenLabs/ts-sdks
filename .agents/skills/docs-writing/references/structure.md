# Documentation Directory Structure

## Content Layout

All documentation lives in `packages/docs/content/`, organized by package:

```
packages/docs/content/
├── sui/                    # @mysten/sui — Core SDK
│   ├── meta.json           # Root section (root: true)
│   ├── index.mdx           # Package overview
│   ├── install.mdx
│   ├── clients/            # Subsection
│   │   ├── meta.json       # Subsection navigation
│   │   ├── index.mdx
│   │   ├── core.mdx
│   │   ├── grpc.mdx
│   │   ├── graphql.mdx
│   │   └── json-rpc.mdx
│   ├── transaction-building/
│   ├── cryptography/
│   ├── utils/
│   └── migrations/
├── dapp-kit/               # @mysten/dapp-kit-core + react
├── kiosk/                  # @mysten/kiosk
├── payment-kit/            # @mysten/payment-kit
├── walrus/                 # @mysten/walrus
├── seal/                   # @mysten/seal
├── zksend/                 # @mysten/zksend
├── bcs/                    # @mysten/bcs
├── codegen/                # TypeScript codegen
├── slush-wallet/           # Slush wallet integration
└── api-reference/          # Auto-generated (excluded from LLM docs)
```

## meta.json Format

### Root Package meta.json

Every top-level package directory must have a `meta.json` with `root: true`:

```json
{
	"root": true,
	"title": "Sui SDK",
	"description": "TypeScript interfaces for Sui",
	"pages": ["index", "install", "hello-sui", "clients", "transaction-building", "cryptography"]
}
```

Fields:

- `root` (required for top-level): Set to `true` to mark as a root section
- `title` (required): Display name for the section
- `description` (recommended): One-line description used in LLM index
- `pages` (required): Ordered array of page names (without `.mdx` extension) and subdirectory names

### Subsection meta.json

Subdirectories can have their own `meta.json`:

```json
{
	"title": "Clients",
	"pages": ["index", "core", "grpc", "graphql", "json-rpc"]
}
```

Subsection meta.json does not have `root: true`.

### Special page names

- `"..."` — Fumadocs rest pages marker. Includes all remaining MDX files not explicitly listed. Used sparingly.
- `"index"` — Maps to `index.mdx` in the directory. Appears first in navigation.

### Pages array ordering

The `pages` array controls navigation order. Put the most important/foundational pages first:

1. `index` (overview/introduction)
2. Getting started / setup pages
3. Core concepts
4. Advanced topics
5. Migration guides (last)

## Build Artifacts

Generated by `pnpm --filter @mysten/docs build:docs`:

```
packages/docs/dist/
├── llms-index.md           # Routing index with all pages and descriptions
├── recommended-context.md  # AGENTS.md snippet for downstream apps
├── sui/                    # Flat markdown mirrors content/ structure
│   ├── index.md
│   ├── clients/
│   │   ├── index.md
│   │   ├── core.md
│   │   └── ...
│   └── ...
├── dapp-kit/
└── ...
```

The `dist/` directory is generated at build time (not committed). After modifying content, regenerate to verify:

```npm
pnpm --filter @mysten/docs build:docs
```

## Adding a New Package Section

1. Create the directory: `packages/docs/content/my-package/`
2. Create `meta.json` with `root: true`, `title`, `description`, and `pages`
3. Create `index.mdx` with `title` and `description` frontmatter
4. Add additional pages as `.mdx` files
5. List all pages in `meta.json` `pages` array
6. Regenerate: `pnpm --filter @mysten/docs build:docs`
7. Validate: `pnpm --filter @mysten/docs validate-docs`

## Adding a Subsection

1. Create the subdirectory within an existing package directory
2. Create `meta.json` with `title` and `pages`
3. Add the subdirectory name to the parent `meta.json` `pages` array
4. Create MDX files for each page listed in `pages`
5. Regenerate and validate
